FAQ

Common questions.

Does Claux see my code or prompts?+

No — and this is a hard architectural boundary, not a policy one. Claux reads only the metadata that Claude Code writes to your local ~/.claude/ session logs: token counts, model names, timestamps, cost estimates, and session identifiers. Your actual code, prompts, and responses never leave your machine and are never read by Claux. The CLAUDE.md generator scans your project structure and file types locally, and the generated config is the only thing that gets processed. If you ever want to verify this, Claux is fully inspectable — no obfuscated binaries.

Does it work with Claude Pro and Max?+

Yes. Claux works with Claude Pro, Max, and direct API access. It reads from the ~/.claude/ session logs that Claude Code writes regardless of your subscription tier. The dashboard, cost tracking, context health monitoring, and CLAUDE.md generator all work the same way across all tiers. The one difference: API users get a token-level cost breakdown per model, while Pro and Max users see session-level aggregates since Anthropic doesn't expose per-token billing for those plans.

Will Anthropic shut this down?+

This is the most common question we get, and the short answer is no — Claux operates entirely within the boundaries Anthropic has defined. We read from ~/.claude/ session logs that Claude Code writes to your local machine. These are your own files, in your own filesystem, and reading them doesn't violate any API terms, usage policies, or licensing agreements. Anthropic has publicly encouraged the developer ecosystem to build tooling around Claude Code. We're not scraping their API, reverse-engineering their models, or accessing anything we're not supposed to. Several Anthropic employees are beta users.

How is this different from ccusage or Nimbalyst?+

ccusage is a lightweight CLI for checking your token usage — excellent for what it does, but it's a single instrument, not a cockpit. Nimbalyst focuses on prompt testing and evaluation. Claux is the only tool that combines real-time observability (live session dashboard, cost ticker, context health), optimisation (CLAUDE.md generation, quality scoring, smart alerts), and structured education (Learning Paths graded against your own data) in a single platform. Think of it as the difference between a speedometer and a full dashboard — you need to know more than just how fast you're going.

How does the CLAUDE.md quality scoring work?+

Claux evaluates your CLAUDE.md against a rubric of 14 criteria, grouped into four categories: context clarity (does it accurately describe your project?), instruction specificity (are the behavioral rules precise enough to be actionable?), consistency (do any instructions contradict each other?), and completeness (are there common Claude failure modes your config doesn't address?). Each criterion is weighted, and you get a score from 0 to 100 with a line-by-line breakdown explaining every deduction. The rubric is updated quarterly as we learn from aggregate anonymised patterns across thousands of user sessions.

Can I use Claux with the Claude API directly, not just Claude Code?+

Yes, with a small amount of setup. Claude Code writes session logs automatically, but if you're calling the API directly you can pipe your usage data to Claux using our lightweight SDK wrapper, which adds two lines to your existing API calls. This gives you the same real-time dashboard, cost tracking, and session analytics as Claude Code users. The CLAUDE.md generator and Learning Paths work independently of how you access Claude — they're not tied to the session log format.

Does Claux work on Windows and Linux?+

Yes. The Claux CLI is cross-platform and tested on macOS, Windows (including WSL), and Linux. The dashboard runs in your browser, so it works anywhere. The only dependency is Node.js 18+. The ~/.claude/ session log format is identical across all platforms since Claude Code uses the same log structure everywhere, so all features work the same regardless of your OS. Installation takes under two minutes on any platform.

What if I cancel?+

If you cancel your Pro or Team subscription, your account immediately drops back to the Free tier — you don't lose access to Claux, you just lose the paid features. All your session history, generated CLAUDE.md files, and Learning Path progress remain accessible for 30 days, during which you can export everything in JSON or CSV format. After 30 days, data associated with paid features is deleted. Free tier data is retained as long as your account is active. There are no cancellation fees and no questions asked.
